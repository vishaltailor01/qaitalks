
# QAi Talks Design System Specification

**Last Updated:** February 18, 2026

## 01 // DESIGN PHILOSOPHY
The "Drafting Table" aesthetic defines the QAi interface. It is a clinical, precise, and authoritative interface mimicking a high-level engineering project manual.

- Content is organized in technical columns with `#0A2540` 1px borders.
- Whitespace is functional, not decorative.
- The interface prioritizes density and clarity over softness.

## 02 // TYPOGRAPHY
- **Primary Sans:** Inter (Headings, Body, Labels)
- **Technical Mono:** JetBrains Mono (Code, Specs, Meta)

## 03 // COLOUR PROTOCOL
| Role         | Color Name    | Hex       | Usage                        |
| ------------ | ------------ | --------- | ---------------------------- |
| Base         | Paper White  | #F0F0FC   | Background/canvas            |
| Ink          | Deep Navy    | #0A2540   | Text, borders, structure     |
| Accent       | Electric Cyan| #00C4FF   | Active states, highlights    |
| Main         | Signal Yellow| #FFB600   | Alerts, caution areas        |

## 04 // COMPONENT DEFINITIONS
### 04.1 INTERACTIVE ELEMENTS
- Buttons: e.g., "Initialize Process", "View Documentation"
  - Height: 40px
  - Radius: 2px
  - Stroke: 1px #0A2540

### 04.2 DATA ENTRY
- System parameter fields (e.g., `Configuration_V2`)
- Clear input styling, technical mono font for values

## 05 // ANNOTATION LOGIC
The "Human Layer" mimics manual markups on a physical blueprint. These elements break the grid rigidity (e.g., "Needs Review" tags, padding checks).

---

### BASE UNITS
- Base unit: 8px
- Column gutter: 24px
- Corner radius: 2px
- Border weight: 1px
